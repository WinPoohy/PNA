Отчет по лабораторной работе №6

Тема: Промисы, Fetch и сборка клиентской части через Vite

1. Цель работы

1.  Модернизировать сетевое взаимодействие фронтенда: заменить устаревшие методы
    на современный стандарт fetch и асинхронный синтаксис async/await.
2.  Реализовать промышленную сборку проекта (bundling) с помощью Vite.
3.  Настроить серверную часть (Express) на раздачу собранного приложения в
    качестве статического контента, тем самым решив проблему CORS на
    архитектурном уровне.

2. Ход работы

Часть 1. Переход на Fetch и Async/Await

В ходе работы сетевой слой был полностью переработан. Мы отказались от
архитектуры коллбэков (ЛР №5) в пользу цепочек промисов. Это позволило
упростить чтение кода и обработку ошибок.

Пример реализации асинхронного метода получения данных в
pages/design-list/index.js:

async getData(query = "") {
    try {
        // fetch возвращает промис, ожидание через await
        const url = query ? `/stocks?title=${query}` : '/stocks';
        const response = await fetch(url);

        // Преобразование ответа в JSON также требует await
        return await response.json();
    } catch (error) {
        console.error("Ошибка при запросе к серверу МГТУ:", error);
        return [];
    }
}

Часть 2. Сборка клиентской части через Vite

Для оптимизации проекта был использован сборщик Vite.

1.  Была настроена конфигурация vite.config.js, указывающая на входные точки
    всех HTML-страниц (главная, дизайн, калькулятор).
2.  Выполнена команда npm run build, в результате чего все модули проекта были
    упакованы и минифицированы в директорию dist/.

Файл package.json с настроенными скриптами:

{
  "scripts": {
    "start": "node src/index.js",
    "build": "vite build",
    "dev": "vite"
  }
}

Часть 3. Интеграция с Backend (Static Assets)

В файле src/index.js бэкенда была настроена раздача статических файлов. Теперь
сервер не только работает как API, но и выступает хостингом для файлов
фронтенда.

Настройка Express для раздачи сборки:

// Подключение middleware для папки со сборкой
app.use(express.static(path.join(__dirname, '..', 'dist')));

// Маршрут для обработки навигации (SPA-подход)
app.get(/^(?!\/stocks).+/, (req, res) => {
    res.sendFile(path.join(__dirname, '..', 'dist', 'index.html'));
});

3. Решение специфических проблем

В процессе настройки сервера на Express 5 возникла проблема с обработкой
универсального пути через символ * (ошибка PathError). Проблема была
решена заменой синтаксиса маршрутизации на регулярные выражения, разрешенные
в новой версии фреймворка, что обеспечило стабильную работу переходов между
страницами дизайна и калькулятора.

5. Контрольные вопросы

1.  Что такое Promise? — Это объект, представляющий результат асинхронной
    операции, который будет доступен в будущем. Имеет три состояния:
    pending, fulfilled, rejected.
2.  Что такое fetch? — Современный программный интерфейс в браузере для
    выполнения HTTP-запросов, полностью основанный на промисах.
3.  Что такое async/await? — Специальный синтаксис, позволяющий писать
    асинхронный код в стиле последовательного (синхронного), что
    значительно повышает его читаемость.
4.  Что такое bundler? — Сборщик (Vite/Webpack), который собирает зависимости,
    стили и модули проекта в компактные и быстрые для загрузки файлы (бандлы).
